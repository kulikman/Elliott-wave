# Elliott Wave Brain — Pine Script индикаторы

Индикаторы для TradingView, реализующие методологию Нили (NeoWave) уровень за уровнем.
Каждый уровень опирается на правила из `brain-output/indicator-spec/spec_vN.json`.

## Принцип построения

Pine Script v6 **не имеет рекурсии** и имеет лимиты на циклы/массивы, поэтому
полный рекурсивный волновой счёт Нили реализуется не «в лоб», а послойно —
снизу вверх. Каждый слой детерминирован и тестируем отдельно.

## Дорожная карта (уровни Нили → модули Pine)

Источник правил: `brain-output/indicator-spec/spec_v2.0.json` — 126 правил
с формализованным `constraint` (псевдокод). Каждый Pine-слой кодируется строго
по constraint с трассировкой `// AKU-XXXX`.

**Сделано (Уровень 2 в `ewb_monowaves_mtf.pine`):** для каждой моноволны вычисляется
`m2/m1 → Правило 1-7` (Определитель Правил, AKU-0141) и `m0/m1 → Условие a-f` (AKU-0142),
метка «П{N}{буква}» выводится на графике. Это структурная классификация Нили.

**Сделано (Уровень 3 в `ewb_monowaves_mtf.pine`):** цепочка базовых структур (:5/:3)
сопоставляется скользящим окном со Структурными Сериями — распознаются и подсвечиваются
рамкой Импульс (5-3-5-3-5), Зигзаг (5-3-5), Плоская (3-3-5), Треугольник/Терминал
(3-3-3-3-3). Приоритет 5-элементных шаблонов (правило «Зигзаг или Импульс?» AKU-0166).

**Честные ограничения распознавания (v1):**
- Используется базовая структура :5/:3 (мой классификатор даёт только её). Точные метки
  :F3/:c3/:L5 и позиционные различия требуют логики Индикаторов положения (Гл.3, стр.3-60+,
  ещё не извлечена) → Зигзаг vs хвост Импульса и Треугольник vs Терминал не различаются.
- Правила 6-7 (откат m1 > 161.8%) дают структуру «?» → разрыв в цепочке, фигура не матчится.

**Следующий шаг:** Индикаторы положения (стр.3-60…3-68) для точного присвоения F/L/c
и однозначного различения фигур; затем подтверждение фигур правилами Гл.5-6 (spec готов).

## ewb_monowaves_mtf.pine (Уровень 1)

**Что делает:**
- Идентифицирует моноволны (движения между сменами направления) reversal-ZigZag методом
- Нумерует их `m0, m1, m2...` на текущем ТФ
- Параллельно строит моноволны старшего ТФ (`M0, M1...`) — **MTF-синхронизация**
- Порог разворота настраивается по ATR или в процентах

**Как запустить:**
1. Открой TradingView → Pine Editor
2. Вставь содержимое `ewb_monowaves_mtf.pine`
3. Add to chart
4. Настрой старший ТФ и порог разворота под свой инструмент

**Реализованные правила из базы AKU:**
- **AKU-0003 — Правило подобия и баланса** (verified + formalized): для каждой пары
  смежных моноволн проверяется `min >= max/3` по цене ИЛИ времени. Если правило
  нарушено → жёлтый маркер `≠` на границе: моноволны разного Порядка, кандидат
  на сжатие (compaction). Это мост от Уровня 1 к Уровню 3 (группировка).

**Ограничения v1 (честно):**
- HTF-пивоты привязываются к bar_index закрытия HTF-бара → возможен небольшой сдвиг
- Это Уровень 1: классификация `:3`/`:5` и распознавание фигур — следующие уровни
- Reversal-порог — упрощение строгого определения моноволны Нили; уточняется на Уровне 2
- Правило подобия пока маркируется визуально; автоматическое сжатие — Уровень 3
