## Обязательные правила

1. **Волна 2 не может откатываться более чем на 100% длины Волны 1. Если ценовое движение опускается ниже начала Волны 1, импульсный счёт неверен.** (AKU-0001) 🔴
> «Волна 2 не может превышать исходную точку Волны 1 — то есть откат не может составлять более 100% длины Волны 1.
»
*Источник: neely-mwe-1990, стр. 3-14*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: low(wave_2) > low(wave_1_start)
```

2. **Второй сегмент импульса (волна 2) никогда не может достигать начального уровня первого сегмента (волны 1). Если это происходит, импульсный счёт невозможен.** (AKU-0004) 🔴
> «Этот второй сегмент никогда не может достигать начального уровня первого сегмента.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: # Второй сегмент никогда не может достигать начального уровня первого сегмента
# Начальный уровень первого сегмента = start(wave_1)
# Конец второго сегмента (и любая точка волны 2) не должен достигать этого уровня

if direction == 'up':
  # Импульс вверх: волна 1 идёт вверх, волна 2 корректирует вниз
  # Начальный уровень волны 1 = low точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  low(wave_2) > price(start(wave_1))
elif direction == 'down':
  # Импульс вниз: волна 1 идёт вниз, волна 2 корректирует вверх
  # Начальный уровень волны 1 = high точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  high(wave_2) < price(start(wave_1))

# "никогда не может достигать" означает строгое неравенство:
# касание уровня start(wave_1) также является нарушением
```

3. **Четвёртый сегмент импульса (волна 4) никогда не должен достигать начального уровня третьего сегмента (волны 3).** (AKU-0005) 🔴
> «Четвертый сегмент никогда не должен достигать начального уровня третьего сегмента.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: pattern == 'impulse' AND wave_4_completed
Правило: # Волна 4 никогда не должна достигать начального уровня волны 3
# Начальный уровень волны 3 = конец волны 2 = start(wave_3)

if direction == 'up':
  # В восходящем импульсе волна 4 корректирует вниз,
  # но не должна опуститься до начала волны 3 (или ниже)
  low(wave_4) > start(wave_3)
elif direction == 'down':
  # В нисходящем импульсе волна 4 корректирует вверх,
  # но не должна подняться до начала волны 3 (или выше)
  high(wave_4) < start(wave_3)
```

4. **Волна-а Зигзага не должна откатиться от предыдущей Импульсной волны более высокого Порядка (если такая волна имеется) больше, чем на 61.8%.** (AKU-0014) 🔴
> «Волна-а не должна откатиться от предыдущей Импульсной волны более высокого Порядке (если такая волна имеется) больше, чем на 61.8%
»
*Источник: neely-mwe-1990, стр. 5-22*

```
Когда: # Текущий паттерн — Зигзаг (Zigzag)
# Рассматривается волна-a этого Зигзага
# Существует предыдущая Импульсная волна более высокого Порядка (degree),
# частью коррекции которой является данный Зигзаг
pattern == 'zigzag' AND current_wave == 'a'
AND exists(previous_higher_degree_impulse_wave)
Правило: # Волна-а Зигзага не должна откатить более 61.8% от предыдущей
# Импульсной волны более высокого Порядка
#
# previous_impulse = предыдущая Импульсная волна более высокого Порядка
# retracement = насколько волна-a откатила от этой импульсной волны
#
# Длина предыдущей импульсной волны:
prev_impulse_length = abs(end_price(previous_higher_degree_impulse_wave) - start_price(previous_higher_degree_impulse_wave))

# Длина отката волны-a (волна-a идёт в направлении, противоположном предыдущему импульсу):
wave_a_length = abs(end_price(wave_a) - start_price(wave_a))

# Ограничение:
wave_a_length <= 0.618 * prev_impulse_length
```

5. **После Удлиненного Зигзага рынок должен развернуться и откатиться более чем на 61.8% длины волны-с, до того как конечный уровень волны-с будет преодолен. Если эти условия не выполняются, вероятно, данная фигура Импульс.** (AKU-0023) 🔴
> «После Удлиненного Зигзага рынок должен развернуться и откатиться более чем на 61.8% длины волны-с, до того как конечный уровень волны-с будет преодолен. Если эти условия выполняются, можно предположит...»
*Источник: neely-mwe-1990, стр. 5-27*

```
Когда: pattern == 'zigzag' AND subtype == 'elongated' AND position == 'post_pattern'
Правило: retrace(next_wave) > 0.618 * length(wave_c) BEFORE price passes end(wave_c)
```

6. **Если длина волны-2 слишком велика по сравнению с длиной волны-1, то волна-2 должна сегментировать, чтобы её волна-с не выходила за конечный уровень её волны-а; в противном случае большая волна начаться не сможет.** (AKU-0050) 🔴
> «Это указывает, что волна-2 должна сегментировать, чтобы ее волна-с не выходила за конечный уровень ее волны-а; в противном случае большая волна начаться не сможет.
»
*Источник: neely-mwe-1990, стр. 8-24*

```
Когда: pattern == 'impulse' AND length(wave_2) large_relative_to length(wave_1)
Правило: is_subdivided(wave_2) AND not exceeds(end(wave_2.c), end(wave_2.a))
```

7. **Начальный уровень любой волны, завершающейся в восходящем направлении и обладающей положительным рейтингом Энергии (от +1 до +3), не может быть достигнут следующей за ней волной такой же степени.** (AKU-0059) 🔴

8. **По завершении Терминальной Импульсной фигуры начальный уровень всей Терминальной фигуры должен быть достигнут за время, не превышающее половины её длительности. Если это правило не выполняется вовсе, гипотеза о Терминальной Импульсной фигуре должна быть отвергнута.** (AKU-0060) 🔴
> «по завершении Терминальной Импульсной фигуры реакция должна быть достаточно мощной, а начальный уровень всей Терминальной фигуры должен быть достигнут за время, не превышающее половины ее длительности...»
*Источник: neely-mwe-1990, стр. 10-1*

```
Когда: pattern == 'terminal_impulse' AND position == 'post_pattern'
Правило: reaches(next_move, start_level(terminal)) AND duration(until_reach) <= 0.5 * duration(terminal)
```
