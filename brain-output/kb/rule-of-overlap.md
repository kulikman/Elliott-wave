## Обязательные правила

1. **Волна 2 не может откатываться более чем на 100% длины Волны 1. Если ценовое движение опускается ниже начала Волны 1, импульсный счёт неверен.** (AKU-0001) 🔴
> «Волна 2 не может превышать исходную точку Волны 1 — то есть откат не может составлять более 100% длины Волны 1.
»
*Источник: neely-mwe-1990, стр. 3-14*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: low(wave_2) > low(wave_1_start)
```

2. **Второй сегмент импульса (волна 2) никогда не может достигать начального уровня первого сегмента (волны 1). Если это происходит, импульсный счёт невозможен.** (AKU-0004) 🔴
> «Этот второй сегмент никогда не может достигать начального уровня первого сегмента.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: # Второй сегмент никогда не может достигать начального уровня первого сегмента
# Начальный уровень первого сегмента = start(wave_1)
# Конец второго сегмента (и любая точка волны 2) не должен достигать этого уровня

if direction == 'up':
  # Импульс вверх: волна 1 идёт вверх, волна 2 корректирует вниз
  # Начальный уровень волны 1 = low точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  low(wave_2) > price(start(wave_1))
elif direction == 'down':
  # Импульс вниз: волна 1 идёт вниз, волна 2 корректирует вверх
  # Начальный уровень волны 1 = high точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  high(wave_2) < price(start(wave_1))

# "никогда не может достигать" означает строгое неравенство:
# касание уровня start(wave_1) также является нарушением
```

3. **Ценовые проекции волн 2 и 4 Трендовых импульсных поливолн (или более сложных трендовых импульсных формаций) не могут перекрываться (накладываться) даже частично.** (AKU-0009) 🔴
> «Ценовые проекции волн 2 и 4 Трендовых импульсных поливолн (или более сложных трендовых импульсных формаций) не могут перекрываться (накладываться) даже частично»
*Источник: neely-mwe-1990, стр. 5-5*

```
Когда: # Правило применяется к трендовым импульсным формациям (5-3-5-3-5)
pattern_type == 'trending_impulse'
# Формация содержит волны 1, 2, 3, 4, 5
waves_count == 5
Правило: # "Ценовые проекции" волн 2 и 4 — это вертикальные ценовые диапазоны,
# которые эти волны занимают на графике.

# Определяем ценовой диапазон волны 2:
wave2_price_min = min(start_price(wave_2), end_price(wave_2))
wave2_price_max = max(start_price(wave_2), end_price(wave_2))

# Определяем ценовой диапазон волны 4:
wave4_price_min = min(start_price(wave_4), end_price(wave_4))
wave4_price_max = max(start_price(wave_4), end_price(wave_4))

# Ценовые проекции волн 2 и 4 НЕ должны перекрываться даже частично.
# Перекрытие существует, если один диапазон заходит в другой:
# overlap = max(wave2_price_min, wave4_price_min) < min(wave2_price_max, wave4_price_max)

# Правило (mandatory): перекрытия быть НЕ должно
overlap_exists = max(wave2_price_min, wave4_price_min) < min(wave2_price_max, wave4_price_max)

REQUIRE overlap_exists == false

# Эквивалентная запись:
# Для бычьего тренда: wave4_price_min >= wave2_price_max
# Для медвежьего тренда: wave4_price_max <= wave2_price_min
#
# Обобщённо (без привязки к направлению):
# wave4_price_min >= wave2_price_max  OR  wave4_price_max <= wave2_price_min
```

4. **Ценовые проекции длин волн 2 и 4 Терминальных Импульсов должны частично перекрываться. Это обязательное условие для Терминального Импульса.** (AKU-0010) 🔴
> «ценовые проекции длин волн 2 и 4 Терминальных Импульсов должны частично перекрываться»
*Источник: neely-mwe-1990, стр. 5-5*

```
Когда: # Правило применяется к Терминальным Импульсам (Ending Diagonals)
pattern == 'terminal_impulse'  # aka ending_diagonal, structure 3-3-3-3-3
Правило: # "Ценовые проекции длин волн 2 и 4" означают вертикальные (ценовые) диапазоны,
# покрываемые волнами 2 и 4. Эти диапазоны должны частично перекрываться.

# Определяем ценовой диапазон волны 2
wave2_high = max(price_at_start(wave_2), price_at_end(wave_2))
wave2_low  = min(price_at_start(wave_2), price_at_end(wave_2))

# Определяем ценовой диапазон волны 4
wave4_high = max(price_at_start(wave_4), price_at_end(wave_4))
wave4_low  = min(price_at_start(wave_4), price_at_end(wave_4))

# Диапазоны должны частично перекрываться (overlap > 0)
overlap = min(wave2_high, wave4_high) - max(wave2_low, wave4_low)

overlap > 0  # MANDATORY: перекрытие должно существовать
```
