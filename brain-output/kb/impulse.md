# Импульс — Knowledge Base

> Топик: `impulse` | AKU verified: 47 | Источники: neely-mwe-1990: 47

## Обязательные правила

1. **Волна 2 не может откатываться более чем на 100% длины Волны 1. Если ценовое движение опускается ниже начала Волны 1, импульсный счёт неверен.** (AKU-0001) 🔴
> «Волна 2 не может превышать исходную точку Волны 1 — то есть откат не может составлять более 100% длины Волны 1.
»
*Источник: neely-mwe-1990, стр. 3-14*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: low(wave_2) > low(wave_1_start)
```

2. **Волна 3 не может быть самой короткой из волн 1, 3 и 5. Если волна 3 короче обеих — импульсный счёт неверен.** (AKU-0002) 🔴
> «Волна 3 не может быть самой короткой волной в импульсной последовательности из волн 1, 3 и 5.
»
*Источник: neely-mwe-1990, стр. 3-15*

```
Когда: pattern == 'impulse'
Правило: length(wave_3) > min(length(wave_1), length(wave_5))
```

3. **Второй сегмент импульса (волна 2) никогда не может достигать начального уровня первого сегмента (волны 1). Если это происходит, импульсный счёт невозможен.** (AKU-0004) 🔴
> «Этот второй сегмент никогда не может достигать начального уровня первого сегмента.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: pattern == 'impulse' AND current_wave == 2
Правило: # Второй сегмент никогда не может достигать начального уровня первого сегмента
# Начальный уровень первого сегмента = start(wave_1)
# Конец второго сегмента (и любая точка волны 2) не должен достигать этого уровня

if direction == 'up':
  # Импульс вверх: волна 1 идёт вверх, волна 2 корректирует вниз
  # Начальный уровень волны 1 = low точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  low(wave_2) > price(start(wave_1))
elif direction == 'down':
  # Импульс вниз: волна 1 идёт вниз, волна 2 корректирует вверх
  # Начальный уровень волны 1 = high точка старта волны 1
  # Волна 2 не должна достигать (или пробивать) этот уровень
  high(wave_2) < price(start(wave_1))

# "никогда не может достигать" означает строгое неравенство:
# касание уровня start(wave_1) также является нарушением
```

4. **Четвёртый сегмент импульса (волна 4) никогда не должен достигать начального уровня третьего сегмента (волны 3).** (AKU-0005) 🔴
> «Четвертый сегмент никогда не должен достигать начального уровня третьего сегмента.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: pattern == 'impulse' AND wave_4_completed
Правило: # Волна 4 никогда не должна достигать начального уровня волны 3
# Начальный уровень волны 3 = конец волны 2 = start(wave_3)

if direction == 'up':
  # В восходящем импульсе волна 4 корректирует вниз,
  # но не должна опуститься до начала волны 3 (или ниже)
  low(wave_4) > start(wave_3)
elif direction == 'down':
  # В нисходящем импульсе волна 4 корректирует вверх,
  # но не должна подняться до начала волны 3 (или выше)
  high(wave_4) < start(wave_3)
```

5. **Пятый сегмент импульса никогда не должен быть меньше 38.2% длины четвёртого сегмента (в ценовом отношении). Когда пятый сегмент короче четвёртого, он называется «Неудавшейся» волной.** (AKU-0006) 🔴
> «Пятый сегмент будет почти, всегда длиннее четвертого, но никогда не должен быть меньше 38.2% длины четвертого (в ценовом отношении).»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: # Правило применяется к импульсной структуре, когда определены сегменты 4 и 5
pattern == 'impulse' AND wave_5.is_completed
Правило: # Длина сегмента в ценовом отношении (вертикальное расстояние)
price_length_wave5 = abs(wave_5.end - wave_5.start)
price_length_wave4 = abs(wave_4.end - wave_4.start)

# Mandatory: пятый сегмент никогда не должен быть меньше 38.2% длины четвёртого
price_length_wave5 >= 0.382 * price_length_wave4

# Дополнительная классификация (информационная, не constraint):
# IF price_length_wave5 < price_length_wave4 THEN wave_5.label = "failure" (Неудавшаяся волна)
```

6. **Третий сегмент импульса никогда не может быть самым коротким из трёх трендовых сегментов (1, 3 и 5) при измерении вертикальных ценовых расстояний. Нарушение этого правила означает, что фигура не является импульсом.** (AKU-0007) 🔴
> «третий не обязательно должен быть самым длинным, но он никогда не может быть самым коротким из трех этих сегментов.»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: # Правило применяется при проверке любой фигуры на импульсность
pattern == 'impulse'
Правило: # Измеряем вертикальные ценовые расстояния (длины) трёх трендовых сегментов
len_w1 = abs(price_end(wave_1) - price_start(wave_1))
len_w3 = abs(price_end(wave_3) - price_start(wave_3))
len_w5 = abs(price_end(wave_5) - price_start(wave_5))

# Третий сегмент никогда не может быть самым коротким из трёх
# Т.е. он должен быть строго больше хотя бы одного из двух других
# (эквивалентно: он НЕ является единственным минимумом или одним из минимумов)
NOT (len_w3 <= len_w1 AND len_w3 <= len_w5)

# Более явная запись:
# len_w3 > min(len_w1, len_w5)

# Если условие нарушено — фигура не является импульсом
```

7. **Если хотя бы одно из семи основных правил импульса не выполняется, анализируемая рыночная активность должна считаться Коррективной, а не Импульсной.** (AKU-0008) 🔴
> «Если хотя бы одно из перечисленных выше правил не выполняется, анализируемая рыночная активность по своему характеру должна считаться Коррективной, а не Импульсной»
*Источник: neely-mwe-1990, стр. 5-1*

```
Когда: # Применяется при валидации любой кандидатной импульсной структуры
# (группа из 5 сегментов, претендующая на статус Impulse)
candidate_pattern == 'impulse'
Правило: # Все семь основных правил импульса должны выполняться одновременно.
# Если хотя бы одно нарушено — паттерн НЕ является импульсом.

# Ссылки на связанные AKU с основными правилами импульса:
# AKU-0004, AKU-0005, AKU-0006, AKU-0007 и остальные из семи правил

rule_results = [
  check_rule_1(waves),  # (правило 1 из 7)
  check_rule_2(waves),  # (правило 2 из 7)
  check_rule_3(waves),  # (правило 3 из 7)
  check_rule_4(waves),  # (правило 4 из 7) — например AKU-0004
  check_rule_5(waves),  # (правило 5 из 7) — например AKU-0005
  check_rule_6(waves),  # (правило 6 из 7) — например AKU-0006
  check_rule_7(waves),  # (правило 7 из 7) — например AKU-0007: wave3 не самая короткая
]

all_rules_passed = ALL(rule_results)

IF NOT all_rules_passed:
  pattern_classification = 'corrective'   # НЕ импульс
  is_valid_impulse = false
ELSE:
  # Все правила выполнены — паттерн может быть импульсом
  is_valid_impulse = true
```

8. **Ценовые проекции волн 2 и 4 Трендовых импульсных поливолн (или более сложных трендовых импульсных формаций) не могут перекрываться (накладываться) даже частично.** (AKU-0009) 🔴
> «Ценовые проекции волн 2 и 4 Трендовых импульсных поливолн (или более сложных трендовых импульсных формаций) не могут перекрываться (накладываться) даже частично»
*Источник: neely-mwe-1990, стр. 5-5*

```
Когда: # Правило применяется к трендовым импульсным формациям (5-3-5-3-5)
pattern_type == 'trending_impulse'
# Формация содержит волны 1, 2, 3, 4, 5
waves_count == 5
Правило: # "Ценовые проекции" волн 2 и 4 — это вертикальные ценовые диапазоны,
# которые эти волны занимают на графике.

# Определяем ценовой диапазон волны 2:
wave2_price_min = min(start_price(wave_2), end_price(wave_2))
wave2_price_max = max(start_price(wave_2), end_price(wave_2))

# Определяем ценовой диапазон волны 4:
wave4_price_min = min(start_price(wave_4), end_price(wave_4))
wave4_price_max = max(start_price(wave_4), end_price(wave_4))

# Ценовые проекции волн 2 и 4 НЕ должны перекрываться даже частично.
# Перекрытие существует, если один диапазон заходит в другой:
# overlap = max(wave2_price_min, wave4_price_min) < min(wave2_price_max, wave4_price_max)

# Правило (mandatory): перекрытия быть НЕ должно
overlap_exists = max(wave2_price_min, wave4_price_min) < min(wave2_price_max, wave4_price_max)

REQUIRE overlap_exists == false

# Эквивалентная запись:
# Для бычьего тренда: wave4_price_min >= wave2_price_max
# Для медвежьего тренда: wave4_price_max <= wave2_price_min
#
# Обобщённо (без привязки к направлению):
# wave4_price_min >= wave2_price_max  OR  wave4_price_max <= wave2_price_min
```

9. **Ни одна часть волн 1 или 3 не должна пересекать действительную линию тренда 0-2.** (AKU-0011) 🔴
> «ни одна часть волн 1 или 3 не должна пересекать действительную линию тренда 0-2.»
*Источник: neely-mwe-1990, стр. 5-6*

```
Когда: # Применяется к импульсной фигуре, когда определены точки 0, 1, 2, 3
# и можно построить линию тренда через точки 0 и 2
pattern == 'impulse'
AND wave_0 IS defined
AND wave_1 IS defined
AND wave_2 IS defined
AND wave_3 IS defined
Правило: # Построить действительную линию тренда (base channel line) через точки 0 и 2
# Точка 0 = начало волны 1, Точка 2 = конец волны 2
trendline_0_2 = line_through(point_0, point_2)

# Для каждого ценового бара внутри волны 1:
#   ни один бар не должен пересекать (пробивать) линию тренда 0-2
FOR each bar IN wave_1.all_bars:
  IF direction == 'up':
    # В бычьем импульсе линия 0-2 проходит по низам, 
    # ни один low бара волны 1 не должен быть ниже значения линии тренда в этот момент
    bar.low >= trendline_0_2.value_at(bar.time)
  ELIF direction == 'down':
    # В медвежьем импульсе линия 0-2 проходит по верхам,
    # ни один high бара волны 1 не должен быть выше значения линии тренда в этот момент
    bar.high <= trendline_0_2.value_at(bar.time)

# Для каждого ценового бара внутри волны 3:
#   ни один бар не должен пересекать (пробивать) линию тренда 0-2
FOR each bar IN wave_3.all_bars:
  IF direction == 'up':
    bar.low >= trendline_0_2.value_at(bar.time)
  ELIF direction == 'down':
    bar.high <= trendline_0_2.value_at(bar.time)
```

10. **Для квалификации самой длинной волны импульсной группы в качестве Растянутой, её ценовая длина должна быть не меньше 161,8% второй по длине волны этой группы. Несоблюдение этого правила практически гарантирует коррективный характер анализируемой группы.** (AKU-0012) 🔴
> «Чтобы квалифицировать самую длинную волну в качестве Растянутой, ее ценовая длина должна быть не меньше 161,8% второй по длине волны этой группы. Несоблюдение любого из представленных до настоящего мо...»
*Источник: neely-mwe-1990, стр. 5-2*

```
Когда: # Правило применяется при проверке группы волн на импульсный характер
# Группа состоит из 5 волн (wave_1, wave_2, wave_3, wave_4, wave_5)
pattern_candidate == 'impulse'
Правило: # Вычисляем ценовые длины действующих (импульсных) волн группы
price_lengths = [
  price_length(wave_1),
  price_length(wave_3),
  price_length(wave_5)
]

# Сортируем по убыванию
sorted_lengths = sort_descending(price_lengths)

longest = sorted_lengths[0]   # самая длинная волна
second_longest = sorted_lengths[1]  # вторая по длине волна

# Самая длинная волна должна быть не меньше 161.8% второй по длине
longest >= second_longest * 1.618

# Если условие НЕ выполнено — группа практически гарантированно
# является коррективной, а не импульсной
# IF NOT (longest >= second_longest * 1.618):
#     pattern_candidate is likely corrective (fails impulse qualification)
```

11. **Для продолжения исследования группы волн в качестве возможного импульса между волнами 2 и 4 должно наблюдаться чередование значений как минимум одного из параметров: цена, время, сила отката, сложность или строение.** (AKU-0013) 🔴
> «Для продолжения исследования группы волн в качестве возможного импульса между волнами 2 и 4 должно наблюдаться чередование значений как минимум одного из описанных выше параметров.»
*Источник: neely-mwe-1990, стр. 5-3*

```
Когда: # Применяется при проверке группы волн как кандидата на импульс
pattern_candidate == 'impulse'
AND wave_2 IS defined
AND wave_4 IS defined
Правило: # Параметры чередования между волнами 2 и 4:

# A. Цена (длина волны в пунктах)
price_alternation = (price_length(wave_2) != price_length(wave_4))
# Существенное различие в абсолютной длине (амплитуде) волн 2 и 4

# B. Время (длительность волны)
time_alternation = (duration(wave_2) != duration(wave_4))
# Существенное различие в продолжительности волн 2 и 4

# C. Сила отката (процент коррекции предшествующей волны)
retracement_w2 = price_length(wave_2) / price_length(wave_1)
retracement_w4 = price_length(wave_4) / price_length(wave_3)
severity_alternation = (retracement_w2 != retracement_w4)
# Существенное различие в глубине отката волн 2 и 4

# D. Сложность (количество составляющих подволн)
complexity_alternation = (subwave_count(wave_2) != subwave_count(wave_4))
# Различие в количестве внутренних сегментов

# E. Строение (тип коррекционной фигуры)
construction_alternation = (pattern_type(wave_2) != pattern_type(wave_4))
# Например: wave_2 == 'zigzag' AND wave_4 == 'flat', или наоборот

# Обязательное условие: хотя бы один параметр должен чередоваться
alternation_exists = (
  price_alternation
  OR time_alternation
  OR severity_alternation
  OR complexity_alternation
  OR construction_alternation
)

# Если чередование отсутствует — группа волн НЕ может рассматриваться
# как возможный импульс (исследование в качестве импульса прекращается)
REQUIRE alternation_exists == true
```

