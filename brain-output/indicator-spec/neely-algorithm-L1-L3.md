## УРОВЕНЬ 1 — Идентификация моноволн (Гл.2)

**Определение** (AKU-0134): моноволна — ценовое движение от одного изменения направления
цены до следующего. Строится по Cash data (не по закрытиям).

```
monowaves = []
для каждого изменения направления цены:
    зафиксировать пивот (жирная точка) на конечных точках движения
    monowave[i] = отрезок между пивотами m[i-1].end и m[i].end
```
Каждой моноволне измеряется длина по ЦЕНЕ и по ВРЕМЕНИ (конечные точки — точки отсчёта).

Для текущей моноволны **m1** берутся соседние: m0 (предыдущая), m2 (следующая), m3 (через одну).

rule = 1 if r<0.382 else 2 if r<0.618 else 3 if |r-0.618|<=0.04*0.618

### Шаг 2.4 — Структурный список → метка (AKU-0148..0150, 0167..0170)

| Правило | Структурный список (приоритет слева) | Вероятная метка |
|---|---|---|
| 1 | {:5, (:c3), (x:c3), [:sL3], [:s5]} | **:5** |
| 2 | {:5, (:sL3), [:c3], [:s5]} | **:5** |
| 3 | {:F3, :c3, :s5, :5, (:sL3), [:L5]} | **:F3** |
| 4 | {:F3, :c3, :s5, (:sL3), (x:c3), [:L5]} (по условию) | **:F3** / :c3 |
| 5 | {:F3, :c3, :5, :L5, (:L3)} | **:F3** |
| 6 | любая (→ Индикаторы положения) | ? |
| 7 | любая (→ Индикаторы положения) | ? |

**Семантика скобок:** без скобок — обычная вероятность; ( ) — низкая; [ ] — крайне низкая.

**Синтез:** малый откат m1 (Пр.1-2) → m1 импульсна `:5`; средний (Пр.3-5) → коррективна `:F3`;
большой (Пр.6-7) → неоднозначно, нужны Индикаторы положения.

### Конвенции измерений (AKU)
- Допуск Фибоначчи **±4%** (61.8% означает 58–66%); «почти»/«близко» = **±10%**.
- По умолчанию измеряется **ценовая** длина.
- Не применять Правила к компактным фигурам, пересекающим свой старт → их база `:3`.

## УРОВЕНЬ 3 — Группировка в фигуры (Гл.4)

### Шаг 3.1 — Выбор группы (AKU-0163)
Брать обособленные группы только из **3 или 5** моноволн (наименьшие по длине/времени первыми).
Поливолна: коррективная ≥3 моноволн, импульсная ≥5 (AKU-0121).

### Шаг 3.2 — Правило подобия и баланса (AKU-0003) — НЕОБХОДИМО для одного Порядка
```
similar(wA, wB) := min(price)/max(price) >= 1/3  OR  min(time)/max(time) >= 1/3
```
Если не выполнено — волны разного Порядка → сжать (compaction) меньшие.

### Шаг 3.3 — Сопоставление со Структурными Сериями (AKU-0159..0163)

| Серия | Шаблон (позиционные метки) | Базовая структура | Фигура |
|---|---|---|---|
| A | :5 -:F3 -:?5 -:F3 -:L5 | 5-3-5-3-5 | **Импульс** |
| B | :5 -:F3 -:?5 | 5-3-5 | **Зигзаг** |
| C | :F3 -:c3 -:?5 | 3-3-5 | **Плоская** |
| D | :F3 -:c3 -:c3 -:?3 -:?3 | 3-3-3-3-3 | **Треугольник** |
| E | :F3 -:c3 -:c3 -:?3 -:L3 | 3-3-3-3-3 | **Терминал** |

**Приоритет:** проверять Импульс (5-эл.) ПЕРЕД Зигзагом — правило «Зигзаг или Импульс?» (AKU-0166):
последние 3 сегмента :5-:F3-:L5 импульса изолированно выглядят как Зигзаг.

### Шаг 3.4 — Индикаторы положения (AKU-0171..0178, 0153/0165 сжатие) — присвоение F/c/L по позиции
- `:F3` — первая тройка (после x:c3 или между :5; две :F3 подряд → вторая = новая фигура).
- `:c3` — центральная: **никогда не первая/последняя**.
- `x:c3` — связка между фигурами; сложность ≤ предыдущей завершённой фигуры; `-`/`++` разделители.
- `:sL3` — предпоследняя: всегда + `:L3`; серия `c3-sL3-3-?`.
- `:L3` — последняя из 5 троек → **Терминал** (если Правила импульса 5-2 выполняются) иначе **Треугольник**.
- `:5`/`:s5`/`:L5` — импульс не-завершающий / особый / завершающий фигуру.
- Разделители: `-` внутри одной фигуры, `++` между фигурами/x-волнами.

### Шаг 3.5 — Сжатие (compaction) и циклы
Подтверждённая группа сжимается до базовой `:3`/`:5`; границы — жирные точки. Серия без «L»
в конце → сжать в `:3` как часть Нестандартной фигуры. Циклы 1→2→3 до появления `:L3`/`:L5`.
Уровни сложности смежных фигур различаются ≤1 (AKU-0153).

## ИЗВЕСТНЫЕ НЕОДНОЗНАЧНОСТИ (честно)

1. **Зигзаг vs хвост Импульса** (5-3-5): разрешается приоритетом импульсной гипотезы (AKU-0162).
2. **Треугольник vs Терминал** (3-3-3-3-3): различаются только последней меткой (:?3 vs :L3) →
   проверкой Правил построения импульса (AKU-0177).
3. **Правила 6-7** (откат m1 >161.8%): структура m1 неоднозначна без полной последовательности
   Индикаторов положения (стр.3-61) — в Pine помечается `?`.
4. Полное присвоение единой метки в Правилах 1-7 требует сотен вероятностных под-правил
   Неформальной логики (стр.3-32..3-59); здесь зафиксирован детерминированный каркас
   (Правило→Условие→приоритетный кандидат), достаточный для базового распознавания.
