# Probability Model v0

Дата: 2026-06-05

Цель модели: показывать вероятности дальнейшего движения цены акции, чтобы Антон мог принимать решения о покупке, продаже или пропуске сделки. Волновая разметка является входом в вероятностную модель, а не конечным продуктом.

## Источники

- `docs/validation/sprint6-final.md` — финальная проверенная стратегия Sprint 6.7.
- `python/data/trades_sprint6.parquet` — сырой профиль сделок sprint6 по типам фигур, направлениям и таймфреймам.
- `python/data/figures_all_tfs.parquet` — база распознанных фигур для дальнейшей калибровки.

Важно: `trades_sprint6.parquet` и `sprint6-final.md` не являются одной и той же агрегацией. Первый файл полезен для эмпирического профиля исходов, второй — для решения, какие фигуры реально торговать в финальной стратегии.

## Торговое решение v0

В v0 торгуются только фигуры с положительным подтверждённым edge:

| Фигура | Действие | Логика | Статус |
|---|---|---|---|
| `flat` | fade direction | ставка на возврат/ретрейс после плоской коррекции | trade |
| `double_corr` | fade direction | ставка на возврат после двойной коррекции | trade |
| `impulse` | no trade | в sprint6-final показан как шум для стратегии | skip |
| `triangle` | no trade | в sprint6-final показан как шум для стратегии | skip |

Базовое правило: если распознана `flat` или `double_corr`, модель оценивает вероятность сделки против направления фигуры. Если распознан `impulse` или `triangle`, индикатор может показывать фигуру, но не должен выдавать торговый сигнал v0.

## Контракт выхода индикатора

Каждый торговый сигнал должен возвращать не только метку фигуры, но и вероятностный объект:

```json
{
  "pattern": "flat",
  "side": "long",
  "p_up": 0.56,
  "p_down": 0.44,
  "p_trade_win": 0.56,
  "expected_net_return": 0.0063,
  "expected_move_pct": 0.024,
  "confidence": "very_high",
  "sample_size": 531,
  "recommended_action": "buy",
  "entry_zone": "fade_after_confirmed_flat",
  "stop": "figure_amplitude",
  "target": "full_retrace",
  "source_model_version": "probability-model-v0"
}
```

Минимальные поля для Pine/TradingView:

- `recommended_action`: `buy`, `sell`, `wait`, `skip`.
- `p_up` / `p_down`: вероятность движения вверх/вниз.
- `p_trade_win`: вероятность прибыльной сделки по выбранной стороне.
- `expected_net_return`: ожидаемая доходность после расходов.
- `confidence`: `low`, `medium`, `high`, `very_high`.
- `sample_size`: количество исторических наблюдений для похожего класса.
- `risk_box`: вход, стоп, цель, invalidation.

## Формула v0

1. Core распознаёт фигуру и направление.
2. Если фигура `flat`, сигнал = fade direction, TP = полный ретрейс, SL = амплитуда фигуры, time exit = 20 баров.
3. Если фигура `double_corr`, сигнал = fade direction, TP/SL по той же логике, time exit = 50 баров.
4. Если фигура `impulse` или `triangle`, торговый сигнал v0 = `skip`.
5. Вероятности берутся из калибровочной таблицы по приоритету: `pattern + timeframe + side`, затем `pattern + timeframe`, затем `pattern`.
6. Если `sample_size < 50`, confidence не выше `low` до дополнительной проверки.

## Роль Pool 2 Overlay

Pool 2 должен не заменять Probability Model v0, а улучшать её:

- S-ка, Стэлла, ВАЛ и эллипсы могут стать признаками для повышения/понижения `confidence`.
- Правило 3-х касаний может уточнять точку входа и invalidation.
- Positive lock относится к trade management, а не к первичной вероятности направления.
- PTV/π/√3 остаются experimental и не должны менять `recommended_action` без отдельного backtest.

Главный критерий включения Pool 2 в индикатор: признак должен улучшать прогноз `p_up/p_down`, `p_trade_win`, `expected_net_return` или drawdown, а не просто добавлять больше волновых линий.
