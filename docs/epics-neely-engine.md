# Дорожная карта волнового движка по Нили (epics)

Источник истины: `aku/neely-mwe-1990/` (362 AKU verified+formalized).
Методология Нили — граф зависимостей: уровень N строится на N-1.
Эпики идут строго в этом порядке.

Статус кода на момент составления:
- Python runtime: `monowaves.py` → `rules.py` (classify_pivots) → `figures.py` (гибрид-геометрия) → `confirm.py` → ML → сигнал
- Pine: `ewb_monowaves_mtf.pine` (3353 стр., L1-L10): моноволны, MTF, подтверждение, каналы, фибо, 3-TP лестница, live-гипотеза, HTF W3 bias
- Торговое ядро: фейдит Flat/DoubleCorr; W3 — только bias-слой

Корневая проблема (зафиксирована в шапке `figures.py`): структурный матчинг :3/:5
падает на реальных импульсах, потому что `rule_to_structure` коллапсирует
Правило→одна метка, теряя полные **структурные списки** (AKU-0148-0150, 0167-0170)
и **Индикаторы Положения** (AKU-0171-0178).

## EPIC 0 — Индикаторы Положения и структурные списки
**Тип:** имплементация (AKU уже формализованы, НЕ нужна формализация)
**Книга:** Гл.3 — Логика Структуры, Определитель Правил, Индикаторы Положения
**Зависимости:** нет. Блокирует EPIC 1, 2, 3.

Сейчас `rule_to_structure()` возвращает одну метку (:5/:F3/?), Правило 6-7 → «?».
Надо вернуть полный структурный список с вероятностной семантикой скобок.

**Точные списки из реестра (verbatim AKU):**
| Правило | m2/m1 | Структурный список m1 | AKU |
|---|---|---|---|
| 1 | <38.2% | `{:5, (:c3), (x:c3), [:sL3], [:s5]}` | 0148 |
| 2 | 38.2–61.8% | `{:5, (:sL3), [:c3], [:s5]}` | 0149 |
| 3 | =61.8% | `{:F3, :c3, :s5, :5, (:sL3), [:L5]}` | 0150 |
| 4 | 61.8–100% | по Условию a-e (см. 0167) | 0167 |
| 5 | 100–161.8% | `{:F3, :c3, :5, :L5, (:L3)}` | 0168 |
| 6 | 161.8–261.8% | любая → Индикаторы Положения | 0169 |
| 7 | >261.8% | любая, favor :5/:s5/:L5/:L3/:F3 | 0170 |

Семантика скобок (AKU-0147): без скобок = норма; `()` = ниже вероятность; `[]` = ещё ниже.
Условия a-f (AKU-0142) и Категории i/iii при Правиле 4 (AKU-0143) сужают список.

**Готово когда:**
- `rule_to_structure_list(rule, cond)` → упорядоченный список `(label, weight)` по всем 7 Правилам
- Последовательность реального импульса больше не содержит «?» там, где структура определима
- Юнит-тесты на синтетическом импульсе/зигзаге проходят
- НЕ трогаем `figures.py` пока (это EPIC 1)

**AKU:** 0141-0150, 0157-0158, 0167-0178 (+0147 скобки, 0142-0143 условия)
